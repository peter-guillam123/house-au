House AU
Search Deep Dive Committees Transcripts Estimates

What this is

A federal Australian sister to House (UK), built for Guardian Australia journalists who need to triage Hansard and ParlInfo faster than the official tools allow. Same five-surface shape as House UK, with a fifth dedicated to Senate Estimates — the twice-yearly hearings where senators grill departmental officials over budget allocations, and where most of the genuinely interesting accountability stories surface. There’s no equivalent in UK House because there’s no equivalent in the UK.

Use cases that drove the shape: a defence correspondent reading everything said about AUKUS in a single sitting fortnight; a political reporter tracking how a junior minister has answered Estimates questions over a year; an investigations editor scanning every committee mention of robodebt or sports rorts; a tech reporter pulling every Estimates exchange about content moderation or Online Safety. The main Search spans the lot; Deep Dive turns a term into a party-stacked timeline; Committees navigates inquiries; Transcripts Estimates searches the words inside committee oral evidence.

How it’s made

Vanilla JavaScript on GitHub Pages, no Worker. House UK leans on UK Parliament’s live, CORS-friendly API; aph.gov.au has no equivalent — just per-day Hansard XML files and ParlInfo’s web search. So House AU inverts the architecture: a nightly GitHub Actions job walks the latest sitting days, downloads each Hansard XML, parses it into speaker-attributed segments, and commits a JSON index back to the repo. The browser fetches that JSON once and searches it client-side. No live API dependency, no proxy, no quota concerns. The cost is a one-day lag — yesterday’s Hansard appears in tomorrow’s harvest, which matters less than it sounds because Hansard itself runs about 24 hours behind the chamber anyway.

Built with Claude Code, internal newsroom tool, currently hosted on my personal GitHub. The Parliament of Australia publishes its material under CC-BY-NC-ND 4.0 — non-commercial, no derivatives. Counsel are relaxed about the framing here because this is a research surface for journalists rather than a public republication of a derivative work. The expectation is that this moves to the Guardian stack behind authentication once it’s doing useful work.

What it doesn’t do yet

Everything. This is scaffold mode — the visual shell, the palette and the page structure are in place, but the harvester that actually fills the index hasn’t been built yet. So a search returns nothing, intentionally and clearly, until the next few commits land. The diary below tracks where we are.

Things deliberately on the not-yet-list: alerts, saved searches, cross-state coverage, the New Zealand sister tool somebody will eventually ask about. Federal first; widen the scope only if the narrow version actually gets used.

Diary

May 2026

Scaffold from House UK, palette swapped to AU green and gold

Cloned the four-page structure from House (UK), dropped the Cloudflare Worker, replaced the oxblood accent with Pantone 348C green for focused states and active filters, and reserved Pantone 116C gold strictly as the search-match highlight fill. Gold on cream paper is 1.3:1 contrast — totally illegible as foreground — so it only ever appears as a background under dark text, never as a link or button colour. Looks good in a brand spec, fails the moment a reader needs to read; flag it loudly so I don’t regress to it later.

Reps and Senate replace Commons and Lords on the house chips. Footer attributions point at aph.gov.au and ParlInfo. The data layer is stubbed — every search returns empty results, intentionally, so pages render their natural empty state instead of crashing on UK Parliament endpoints. The console logs a one-line scaffold-mode notice so it’s obvious what’s happening.

Next: the harvester that actually fetches Hansard XML from aph.gov.au and writes the JSON shards the search will read. That’s the first ‘shows real data’ commit and the architectural heart of the divergence from House UK.
