House
Search Deep Dive Committees Transcripts

What this is

A set of four search surfaces over UK parliamentary proceedings, built for journalists who need to triage faster than parliament’s own tools allow. The main Search pulls from four official APIs at once — spoken contributions, written questions and answers, written ministerial statements, and committee debates — and shows the lot in one ranked list. Deep Dive plots a single term as a party-stacked monthly timeline with click-to-filter panels for members, debates and co-occurring phrases. Committees finds inquiries by name and lets you drill into one to search its transcripts directly. And Transcripts searches the actual words inside select-committee oral evidence — the bit Hansard’s API doesn’t expose for search, made possible here by a daily-built static index.

Use cases that drove the shape: a defence correspondent looking for every mention of RAF Lakenheath in the last year; a health editor scanning what’s been said about a specific hospital trust; a political reporter tracking how often a junior minister has spoken on a topic; a tech reporter looking for every committee witness from Meta or Roblox; a feature writer pulling all the times Wendy Williams has been raised in evidence.

How it’s made

Vanilla JavaScript and a small Cloudflare Worker. The frontend on GitHub Pages talks to five parliament APIs via a 50-line Worker proxy — needed because most of them don’t return CORS headers. Transcripts is the exception that drove a real backend addition: a daily GitHub Actions job walks the committees API, fetches each oral evidence transcript, parses it into speaker-attributed segments, and commits a JSON index back to the repo. The browser fetches that file once and searches it entirely client-side. The rolling 90-day window is rebuilt every night; the sealed quarterly archives are built once per quarter and frozen.

Built with Claude Code over a series of conversations. Source is at github.com/peter-guillam123/House. All the data is from parliament under the Open Parliament Licence v3.0.

What it doesn’t do

The transcript archive goes back to the start of 2020 — six years of select-committee evidence, around 3,500 sessions across the quarterly archives. Worth knowing that the July 2024 general election was a hard reset on committee membership and on what was being inquired into, so a search that crosses that line is mixing two different parliaments’ preoccupations. Pre-2020 is feasible to backfill if anyone needs it.

The transcripts come out of parliament’s .docx-to-HTML conversion pipeline, which sometimes sticks single-character spaces inside words (“giraffes” ends up indexed as “gi raf fes”). A search for “RAF” will occasionally false-match a standalone “raf” sitting inside another word. That’s the source data — fixing it without corrupting legitimate text would need OCR-grade reasoning.

Multi-word terms are matched as a phrase, so “RAF Lakenheath” finds that exact phrase rather than RAF and Lakenheath separately. There’s no Boolean syntax (AND/OR/NOT). For more advanced queries on Hansard, fall back to hansard.parliament.uk directly.

Changelog

May 2026

First version live

Four sources, five filters, results sorted newest first. Built in a single planning-and-execution pass with Claude Code. The hard part turned out not to be the APIs — they’re genuinely good — but the fact that two of them don’t return CORS headers, which would have killed a pure static-site build. A 40-line Cloudflare Worker fixes that for less than the cost of a coffee per year.

Things deliberately left out of v1: alerting (email me when X gets mentioned), saved searches, CSV export. Probably the right next things to add if it gets used.

May 2026

Premium pass

Reset the surface around an editorial-archive direction: ivory paper, near-black ink, a single restrained oxblood accent. Source Serif 4 for display, Inter for body, JetBrains Mono for stamps. Sticky search on mobile, collapsible filter shell, generous result rhythm. New favicon and OpenGraph card. Functionality unchanged.

May 2026

Deep Dive on mobile

Tightened the Deep Dive page on phones. The chart was sitting in a padded box and capped at 200px deep, which flattened the party-stacked shape; reduced the inset and let it breathe to 280px. The two leaderboards now show the top five by default with a “Show all” reveal, so the actual contributions list isn’t buried half a screen down.

May 2026

Panels as filters

The leaderboards and the legend on Deep Dive turned out to be better filter surfaces than any panel of dropdowns would have been — they’re already organised by who’s spoken most, which debates are carrying the topic, and which parties are doing the talking. So they now double as filters. Click a member, a debate or a party chip in the legend and the contributions list narrows; clicks AND-combine across the three axes; an active-filter strip sits above the list with one-tap removal. Shareable in the URL.

The chart itself stays as the universe view — the filters narrow the contributions list, not the shape of the conversation. The leaderboards don’t re-rank either, so they stay usable as navigation while filters are active.

Along the way: extracted a shared lozenge style so party tags look the same wherever they appear; nudged the palette (SNP yellow was disappearing on cream paper, Lab/Co-op was bleeding into Lab); and sharpened the dek to make the spoken-contributions-only point properly. Legend chips on desktop now reveal each party’s running total on hover.

May 2026

Quoted searches highlight properly

Searching for “the Guardian” in quotes was finding the right contributions, but the highlights were going wrong in two places. Our snippet highlighter was looking for the literal quoted string in the result text — and the text doesn’t have those quote characters around the phrase, so it found nothing. Hansard’s own ?highlight= parameter on click-through couldn’t parse the quoted form either, and was marking just “the” on its own. Strip the wrapping quotes for highlight purposes; leave the quoted form intact for the API search itself, which uses it to do phrase matching.

May 2026

Form polish, search pulse

Two more polish items. The Deep Dive form now matches Search on mobile — icon + input + button on one row, with the year range on a quiet second row below. Both forms pulse a thin gradient bar at the bottom edge while a search is in flight: quieter than the colour change at submit, more useful on a Deep Dive that can stream for thirty seconds.

May 2026

One shape for every result row

Search and Deep Dive had drifted into different visual idioms for the same job. Search rendered everything in mono caps with party as an inline (LAB) text and House as a tiny mid-line label; Deep Dive used sentence case and a coloured party lozenge. Pulled both onto a single shape: mono-caps date, sentence-case member name, coloured party lozenge, neutral outlined House lozenge. Side effect of the typography change: minister-by-role attributions like ‘The Parliamentary Under-Secretary of State, Ministry of Housing, Communities and Local Government (Baroness Taylor of Stevenage)’ became readable rather than a wall of caps.

May 2026

Three lifts that aren’t in Hansard

A small strategic pass — what can the page do that Hansard’s own search can’t? Three things at once.

An Export button next to the contributions count on both Search and Deep Dive downloads the current filtered set as Markdown — date, member, party, House, debate, full passage text, and a deep link back to Hansard. Drops straight into NotebookLM, Obsidian, or a Google Docs paste. Since the click-to-filter UI already lets you curate by member, debate, party or co-term, the filter system is the research basket; no need for a separate starring affordance.

A new ‘Often mentioned alongside’ panel beside the existing leaderboards on Deep Dive lists the multi-word capitalised phrases that recur across the matching contributions. For Windrush you get Empire Windrush, Wendy Williams, Home Office, Caribbean Community surfacing — a tipsheet in a panel. The first version drowned in noise (Hon Member, Madam Deputy Speaker, Liberal Democrats, ‘Recent Developments’ as a Hansard heading template); a couple of rounds of stopword tightening on real samples cleaned it up. Click a phrase to filter to contributions that mention it, AND-combined with the other filters.

And the stats row picked up small sub-lines under each cell: the peak month’s count, who first mentioned the term, who most recently raised it.

May 2026

Searching the words inside transcripts

The big one for select-committee evidence. Hansard’s API doesn’t expose transcript text for search; the committees API exposes the documents as base64-wrapped HTML but not as searchable content. So a daily GitHub Actions job walks the last 90 days of oral evidence sessions, fetches each transcript, parses it into speaker-attributed segments, and writes evidence-index.json at the repo root. Static-site search, no backend.

The new Transcripts page loads the index on entry and runs queries entirely in the browser. Snippets carry the speaker label, the previous turn for context (so a hit in a witness’s answer comes paired with the question that prompted it), and a text-fragment deep link that opens the published transcript scrolled to the line. Click-to-expand on long prior turns.

The 90-day rolling window keeps the page genuinely current without ballooning the file. To go further back, the page carries a switcher of sealed quarterly archives. Each quarter is built once, frozen, and lived alongside the rolling index. Only one is loaded at a time — keeps memory bounded on phones.

OCR-mangled words from the .docx-converted transcripts (“giraffes” indexing as “gi raf fes”) still show up. That’s the source data — fixing it without corrupting legitimate text would need OCR-grade reasoning. Live with it; flag in this About; move on.

May 2026

Six years of archive, tabular picker, smarter acronyms

The Transcripts page now spans the start of 2020 to the present rolling 90 days — about 3,500 sessions across 25 quarterly archives plus the rolling index. The session counts tell their own political story when you look at the grid: the 13-session 2024 Q3 (election dissolution + recess), the small Q3s every year for summer recess, the COVID scramble in 2020 Q2.

The pill-row layout that worked for two or three quarters would have been unmanageable here, so the picker became a Year × Quarter table with a session count in each cell — readable as a story at a glance, and there’s room to add more years before it strains.

And acronyms: searching ‘SAS’ was matching ‘sas’ inside ‘passage’ and ‘Sasaki’; ‘DEFRA’ was hitting random letter strings. Short all-caps terms (2–8 chars) now match case-sensitively with word boundaries everywhere — Search snippets, Deep Dive snippets, within-inquiry search, and the global Transcripts search. Mixed-case terms keep the existing case-insensitive behaviour. Centralised in one helper, one rule.
